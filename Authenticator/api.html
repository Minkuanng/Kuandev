📘 Tài liệu API đặt hàng & nhập kho

API dành cho website đối tác (đại lý) đặt mua sản phẩm bằng mã shop_id + token được cấp trong trang quản trị (tab API / Đại lý). Mỗi đại lý có số dư riêng, không dùng chung ví người dùng.

Cách tạo API GET / POST từ Vercel

Để có các endpoint API hoạt động, bạn cần deploy repository vercel-api lên Vercel. Dưới đây là hướng dẫn chi tiết từng bước:

1
Tạo repository trên GitHub
Tạo repo mới tên vercel-api và clone về máy.
2
Tạo cấu trúc thư mục
api/ → chứa các file endpoint
lib/ → chứa file cors.js
package.json → quản lý dependencies
3
Tạo file endpoint mẫu
Tạo file api/products.js với nội dung lấy danh sách sản phẩm từ Firebase.
Xem mã mẫu bên dưới.
4
Cấu hình Firebase Admin
Thêm biến môi trường trên Vercel:
FIREBASE_PROJECT_ID, FIREBASE_CLIENT_EMAIL,
FIREBASE_PRIVATE_KEY, FIREBASE_DATABASE_URL
5
Deploy lên Vercel
Import repository vercel-api lên Vercel → Deploy.
Sau khi deploy, bạn sẽ có base URL:
https://vercel-api-mocha-zeta.vercel.app
Base URL mẫu: https://vercel-api-mocha-zeta.vercel.app/api
📌 Thay đổi URL này thành URL thực tế của bạn sau khi deploy.

📄 Mã mẫu cho api/products.js

// api/products.js
import admin from 'firebase-admin';
import { getCorsHeaders, handleCors } from '../lib/cors.js';

// Khởi tạo Firebase Admin
if (!admin.apps.length) {
  try {
    if (process.env.FIREBASE_PROJECT_ID) {
      admin.initializeApp({
        credential: admin.credential.cert({
          projectId: process.env.FIREBASE_PROJECT_ID,
          clientEmail: process.env.FIREBASE_CLIENT_EMAIL,
          privateKey: process.env.FIREBASE_PRIVATE_KEY.replace(/\\n/g, '\n'),
        }),
        databaseURL: process.env.FIREBASE_DATABASE_URL,
      });
    }
  } catch (error) {
    console.error('Lỗi khởi tạo Firebase Admin:', error);
  }
}
const db = admin.database();

export default async function handler(req, res) {
  // Xử lý CORS
  if (req.method === 'OPTIONS') {
    return handleCors(res);
  }

  if (req.method !== 'GET') {
    res.setHeader('Allow', ['GET']);
    return res.status(405).json({ success: false, error: 'method_not_allowed' });
  }

  try {
    const snapshot = await db.ref('products').once('value');
    const productsData = snapshot.val();

    if (!productsData) {
      return res.status(200).json({ success: true, products: [], total: 0 });
    }

    const products = Object.keys(productsData).map(key => {
      const p = productsData[key];
      const dataArray = p.data ? p.data.split('\n').filter(d => d.trim() !== '') : [];
      return {
        product_id: key,
        name: p.name || 'Sản phẩm',
        price: p.price || 0,
        icon: p.icon || '📦',
        image: p.image || '',
        description: p.description || '',
        category: p.category || '',
        stock: dataArray.length,
        sold: p.sold || 0,
      };
    });

    res.status(200).json({ success: true, total: products.length, products });
  } catch (error) {
    console.error('Lỗi:', error);
    res.status(500).json({ success: false, error: 'server_error' });
  }
}
Mẹo: Bạn có thể tạo thêm các file trong api/:
  • api/product/[id].js → Lấy chi tiết sản phẩm
  • api/order.js → Đặt hàng (POST)
  • api/shop/balance.js → Kiểm tra số dư đại lý
  • api/categories.js → Lấy danh sách danh mục

POST /api/order

POST https://vercel-api-mocha-zeta.vercel.app/api/order

Body (JSON)

Trường Kiểu Bắt buộc Mô tả
tokenstring✔Token của đại lý
shop_idstring✔Mã đại lý
product_idstring✔ID sản phẩm (xem tab Kho hàng hoặc GET /api/product/:id)
quantitynumberSố lượng, mặc định 1
input_datastringThông tin đối tác truyền vào (VD: UID, ghi chú)

    "product_id": "-Nabc123xyz",
    "quantity": 1,

GET sản phẩm từ API → Nhập kho

Bạn có thể lấy danh sách sản phẩm từ API bên ngoài (Vercel, Shopify, WooCommerce, ...) và nhập trực tiếp vào kho hàng của KuanZGame.Store bằng tính năng "API Import" trong tab API Import của trang quản trị.

1. Lấy JSON 2. Paste URL 3. Chọn mục 4. Nhập kho

📍 Cách dùng

  1. Chuẩn bị API endpoint trả về danh sách sản phẩm dạng JSON (ví dụ: https://vercel-api-mocha-zeta.vercel.app/api/products)
  2. Vào trang Quản trị → tab API Import
  3. Dán URL API vào ô "Lấy dữ liệu" và nhấn nút
  4. Hệ thống sẽ fetch và preview danh sách sản phẩm
  5. Chọn mục sản phẩm → nhấn "Nhập vào kho"
Định dạng JSON hỗ trợ:
  • { "products": [ { "name": "...", "price": 100000, "data": "..." } ] }
  • [ { "name": "...", "price": 100000, "data": "..." } ]
  • { "data": [ { "name": "...", "price": 100000, "data": "..." } ] }
  • { "success": true, "products": [ ... ] } (từ API của bạn)

Mã lỗi

HTTP error Ý nghĩa
400missing_fieldsThiếu trường bắt buộc
400invalid_quantityquantity không hợp lệ
401invalid_tokenToken sai hoặc không khớp shop_id
403shop_bannedĐại lý đã bị khóa
404shop_not_foundshop_id không tồn tại
404product_not_foundproduct_id không tồn tại
409out_of_stockKhông đủ hàng trong kho
402insufficient_balanceSố dư đại lý không đủ
500server_errorLỗi máy chủ, thử lại sau

Ví dụ gọi bằng JavaScript (Node.js)

// Lấy danh sách sản phẩm
const productsRes = await fetch("https://vercel-api-mocha-zeta.vercel.app/api/products");
const products = await productsRes.json();
console.log(products);

// Đặt hàng
const orderRes = await fetch("https://vercel-api-mocha-zeta.vercel.app/api/order", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    token: "sk_xxx...",
    shop_id: "shop_ab12cd3",
    product_id: "-Nabc123xyz",
    quantity: 1,
    input_data: "user123@game"
  })
});
const order = await orderRes.json();
console.log(order);
Luôn gọi API từ backend của đối tác, không gọi trực tiếp từ trình duyệt khách hàng cuối vì token sẽ bị lộ.

Tóm tắt luồng làm việc

1 Deploy Vercel API → Tạo base URL
2 GET từ API → Lấy JSON sản phẩm
3 Admin preview dữ liệu, chọn mục
4 Nhập vào kho → sản phẩm có sẵn để bán
